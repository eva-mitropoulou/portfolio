Antibody Sequence ML And Existing-Record Prioritization

Public-data antibody sequence workflow for retrospective benchmarking and existing-record prioritization. This is benchmark and triage work, not therapeutic antibody design.

1. Overview

Built a public-data antibody sequence ML workflow for retrospective benchmarking and existing-record prioritization with careful claim boundaries around labels, validation, and OAS background semantics.

2. Scientific And Technical Problem

Public antibody datasets mix assay labels, source records, target annotations, and sequence completeness. The workflow tests whether sequence features and antibody-specific representations can be benchmarked reproducibly without overstating prospective biological utility.

3. Dataset Or System

  • CoV-AbDab-derived public-data workflow.
  • Strict labeled dataset: 5,573 records.
  • Broader prepared-record set: 11,748 records.
  • Paired CDR-annotated subset: 5,092 candidates.
  • OAS background records treated as unknown-target background rather than assayed controls.

4. Methods

Character k-mer TF-IDF baselines, logistic regression, grouped validation, source holdout, source-robust model selection, calibration analysis, CDR annotation, AbLang2-style embeddings, IgBert-family benchmark artifacts where present, OAS background retrieval, and diversity-aware existing-record prioritization.

5. Validation Strategy

Grouped validation is prioritized where group features are meaningful. Random split results are retained as baselines. Matched CDR comparisons compare whole-pair and region-local k-mer features on the same paired subset. Source-holdout validation is used as a skeptical control for source and study effects, and source-robust model selection chose whole_pair_kmer.

8. Limitations

  • Retrospective public labels.
  • Heterogeneous assays and sources.
  • Source and study effects limit cross-source generalization.
  • Existing-record prioritization only.
  • Not a sequence-design, mutation, or optimization workflow.
  • No prospective validation.
  • OAS records are unknown-target background rather than assayed controls.
  • Model scores are prioritization signals, not calibrated prospective neutralisation probabilities.

9. Reproducibility

Public-safe reproduction uses cached summaries and report generation. Full reproduction may require raw public-data downloads, embeddings, model weights, and heavier compute.
